Server-Side Web Scripting

Java Servlets

-Java objects that are intended to play the role of server components in client-server communications.

-Handles client requests by generating responses to such requests.

-Part of the Java EE specification.

What is Java EE?

Java Platform, Enterprise Edition or Java EE is a set of specifications that defines APIs and their interactions.

-Hosted in a 'servlet container' that provides the environment in which the servlet runs, as well as controls the 'servlet lifestyle'.

Servlet Lifestyle

1. Instantiation
Constructor Call -creates an instance of the servlet to service client requests.
2. Initialization
init() method -invoked only once and is intended for any startup initialization code for the servlet.
3. Request Handling
service() method -it is a main method that performs the actual task.
4. Destruction
destroy() method -invoked before the servlet instance is 'unlaoded', and its intended for any 'housekeeping' code that the servlet may need to run.

HTTP Servlet

-A java servlet used to handle HTTP requests and generate responses.

-Hosted in a 'web container', which is a component in a Java 'application server'.

-The service() method call is routed to a doXXX() call, depending on the HTTP request method. (e.g. doGet(), doPost(), doHead(), doPut(), doDelete())

The different doXXX() methods passed two arguments:
1. HTTPServletRequest - object representation of the HTTP request generated by the servlet.
2. HTTPServletResponse - object representation of the HTTP response generated by the servlet and sent back to the requesting servlet.
